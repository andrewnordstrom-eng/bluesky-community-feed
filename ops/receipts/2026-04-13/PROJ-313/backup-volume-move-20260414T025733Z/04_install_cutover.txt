== PROJ-388 current reviewed Bluesky state ==
- daily-backup.sh validates the exact backup mount target with
  findmnt -n -o TARGET --target "$BACKUP_MOUNT_ROOT" before creating
  ${BACKUP_MOUNT_ROOT}/postgres or ${BACKUP_MOUNT_ROOT}/igor/daily.
- daily-backup.sh normalizes trailing slashes in BACKUP_MOUNT_ROOT before
  comparing the expected and actual mount targets.
- bluesky-ops-retention.sh validates the exact mount target before any
  missing-POSTGRES_DIR skip path, so an absent mount fails closed.
- install.sh uses the same exact-target guard before creating backup data
  directories, keeps /opt/backups/daily-backup.sh as the cron-compatible script
  install path, creates /opt/backups as a 0700 script/log directory, and
  pre-creates /opt/backups/backup.log as root:root 0600.
- backup data remains rooted under /mnt/host-backups.
